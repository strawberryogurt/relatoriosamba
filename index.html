Relatório Samba Server

Tauana Auler & Tobias Trein

Instalando o Samba Server:

Primeiro, no terminal, realizamos atualização do apt e instalação do Samba Server com os seguintes comandos respectivamente:
 sudo apt update
 sudo apt install samba

Para testar se a instalação foi concluída com sucesso use o comando:
 whereis samba

Se estiver tudo certo retornará algo parecido com:
 samba: /usr/sbin/samba /usr/lib/samba /etc/samba /usr/share/samba /usr/share/man/man7/samba.7.gz /usr/share/man/man8/samba.8.gz


Configurando o Samba Server:

Nós já criamos anteriormente as pastas gerencia, sistemas e util e agora entraremos na configuração do Samba para definir as permissões quanto a elas.
Entre na configuração com:
 sudo nano /etc/samba/smb.conf
Ao final do arquivo que abrir acrescentamos: as seguintes linhas:
[home]
  path = /home
  read only = no
  browsable = yes
[sistemas]
  path = /sistemas
  read only = no
  browsable = yes
[util]
  path = /util
  read only = no
  browsable = yes
[gerencia]
  path = /gerencia
  read only = no
  browsable = yes

Ao sair do editor usamos para reiniciar o samba o comando:
 sudo service smbd restart

O Samba não utiliza a mesma senha que a máquina, então é necessário definir uma usando o comando:
 sudo smbpasswd -a username

O servidor já está funcionando, podendo ser conferido com o comando:
 sudo service smbd status

Para acessá-lo, digite o caminho no explorador de arquivos do SO:
 smb://ip-do-servidor/pasta

No nosso caso, podemos acessar, por exemplo:
 smb://.10.0.2.15/sistemas
